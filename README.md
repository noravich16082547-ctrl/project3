# DormCRU — เว็บไซต์หาหอพักสำหรับนักศึกษาปี 1 ราชภัฏเชียงราย
### (เวอร์ชัน GitHub + Vercel + Supabase, ไฟล์แบนราบ)

ดูขั้นตอนติดตั้งทั้งหมดใน **`SETUP-SUPABASE.md`** — สร้าง Supabase, รัน `schema.sql`,
เชื่อม GitHub เข้ากับ Vercel, ตั้งบัญชีแอดมินคนแรก

## โครงสร้างผู้ใช้งาน 3 ระดับ
- **นักศึกษา** — สมัครที่แท็บ "สมัครนักศึกษา" ในหน้า `/login` ค้นหา/จอง/บันทึกหอพักที่ถูกใจ
- **เจ้าของหอพัก** — สมัครที่แท็บ "สมัครเจ้าของหอ" ต้องรอแอดมินอนุมัติก่อนจึงเพิ่ม/แก้ไขหอพักได้
- **แอดมิน** — เห็นและจัดการได้ทุกหอพัก/ทุกการจอง + อนุมัติเจ้าของหอพักรายใหม่

## ทำไมเวอร์ชันนี้ถึงต่างจากเวอร์ชันก่อน
เวอร์ชันก่อนหน้าเก็บไฟล์ไว้ในโฟลเดอร์ `assets/` และให้ JavaScript สร้างเนื้อหาเกือบทั้งหมด
ของหน้าเว็บ — พอ deploy ผิด path หรือ Supabase ยังไม่ได้ตั้งค่า หน้าเว็บเลยพังจนขึ้นว่างเปล่า

เวอร์ชันนี้แก้ 3 อย่าง:
1. **ไฟล์แบนราบทั้งหมด** อยู่ที่รากโปรเจค ไม่มีโฟลเดอร์ย่อย ตัดปัญหา path ผิดตอน deploy
2. **`db.js` ไม่มีวันทำให้หน้าเว็บพัง** — เช็คตัวเองก่อนทุกครั้ง ถ้า Supabase ยังไม่พร้อมจะโชว์
   แบนเนอร์เตือนแทนที่จะปล่อยให้ error หยุดทั้งหน้า
3. **หน้ารายละเอียด/จอง/รายการที่ถูกใจ รวมอยู่ใน `index.html` เป็นโมดัล** แทนการแยกเป็นหลายหน้า
   (ลดความเสี่ยงเรื่องลิงก์ระหว่างหน้าเสีย)

## ไฟล์สำคัญ
- `db.js` — **ต้องแก้ `SUPABASE_URL`/`SUPABASE_ANON_KEY` เป็นค่าจริงก่อนใช้งาน**
- `schema.sql` — รันใน Supabase SQL Editor ครั้งเดียวตอนตั้งค่า
- `index.html` — หน้าแรก ค้นหา/กรอง/รายละเอียด/จอง/wishlist ทั้งหมดในหน้าเดียว อัปเดตห้องว่างแบบเรียลไทม์
- `login.html` — เข้าสู่ระบบ + สมัครนักศึกษา + สมัครเจ้าของหอ (3 แท็บ)
- `admin.html` + `admin.js` — หลังบ้านเจ้าของหอพัก/แอดมิน
- `vercel.json` — clean URL ให้ `/login` และ `/admin` ใช้แทน `.html` ได้

## ข้อควรทราบ
- bucket `slips` ตั้งเป็น public เพื่อความง่าย (ไฟล์เข้าถึงได้ถ้ารู้ลิงก์ตรงๆ ชื่อไฟล์สุ่มไม่ให้เดาง่าย)
- ข้อมูลหอพักตัวอย่างมีคำว่า "(ข้อมูลตัวอย่าง)" กำกับไว้ ให้ลบ/แก้ก่อนเปิดใช้งานจริง
- Supabase free tier จะหยุดทำงานชั่วคราวถ้าไม่มีคนใช้เกิน 7 วัน ต้องกด "Restore project" ปลุกเอง


## อัปเดตล่าสุด: ระบบสำหรับใช้งานจริงกับหอพักจริง

**1. ข้อมูลหอพักจริง (ไม่ใช่หอสมมติ)** — ข้อมูลตั้งต้นเปลี่ยนเป็นหอพักที่มีอยู่จริงรอบ มร.ชร. ต.บ้านดู่
(เวสธิมา คอร์ท หน้ามอ, ฮักล้านนา ประตู 2, หอพักสุนทรี, จงไพศาล 2, บุญมณี ฝั่งประตู 3 ฯลฯ)
รวบรวมจากประกาศสาธารณะ ครอบคลุมทั้ง 3 ประตู **แต่ระบบตั้งสถานะ "รอเจ้าของหอยืนยัน" (verified=false)
ให้ทั้งหมด** เพราะราคา/ห้องว่าง/ช่องทางติดต่อจากประกาศอาจไม่เป็นปัจจุบัน — หอที่ยังไม่ยืนยันจะ:
- แสดงป้ายเตือนชัดเจนว่าข้อมูลรอการยืนยัน
- ไม่แสดงจำนวนห้องว่าง (ขึ้น "สอบถามห้องว่างกับหอโดยตรง" แทน)
- เปิดเฉพาะ "ขอนัดดูห้อง/จองคิวไว้ก่อน" — ไม่เปิดรับโอนมัดจำ เพื่อป้องกันการโอนเงินจากข้อมูลที่ไม่ยืนยัน
วิธียืนยัน: เจ้าของหอตัวจริงสมัครบัญชี → แอดมินอนุมัติ → แอดมินเข้าแก้ไขหอนั้นแล้วติ๊ก "ยืนยันข้อมูลแล้ว"
(หอที่เจ้าของเพิ่มเองใหม่จากหลังบ้าน = verified อัตโนมัติ เพราะเป็นข้อมูลจากเจ้าของโดยตรง)

**2. รูปภาพ** — ใช้ภาพถ่ายจริงจากช่างภาพ (คลังภาพ Pexels) ไม่ใช่ภาพ AI แต่ยังเป็น "ภาพตัวอย่างบรรยากาศ"
ไม่ใช่ภาพถ่ายของอาคารแต่ละหลังจริง เพราะการดึงรูปจริงของธุรกิจมาใช้โดยไม่ได้รับอนุญาตทำไม่ได้
— เจ้าของหอลงรูปจริงของตัวเองแทนได้ทันทีในหลังบ้าน (ช่อง "ลิงก์รูปภาพ")

**3. โหมดจองใหม่: จองไว้ก่อน แล้วนัดทำสัญญา** — นักศึกษากดจอง → กรอกเบอร์โทร + วันเวลาที่สะดวก
→ หอเห็นคำขอในหลังบ้านพร้อมเบอร์ติดต่อกลับ → โทร/ไลน์นัดดูห้อง-ทำสัญญา → กดยืนยันในระบบ
(สลิปมัดจำเปลี่ยนเป็น "ไม่บังคับ" — แนบได้เฉพาะหอที่ยืนยันข้อมูลแล้วและตกลงกันแล้ว)

**4. ช่องทางติดต่อหอพัก** — เพิ่มช่อง เบอร์โทร / LINE / Facebook ต่อหอ แสดงเป็นปุ่มกดโทร-เปิดไลน์-เปิดเพจ
ได้ทันทีในหน้ารายละเอียด เจ้าของหอกรอกเองในหลังบ้าน

**5. เครื่องมือเจ้าของหอ** — ปุ่ม +/− ปรับห้องว่างเร็ว (เปิดห้อง/ปิดห้อง) หน้าตารางจัดการ ไม่ต้องเปิดฟอร์ม,
แก้ราคา, ลงรูป, แก้ช่องทางติดต่อ ได้ครบในฟอร์มแก้ไข

เพื่ออัปเกรดตาราง (คนตั้งโปรเจคใหม่ใช้ `schema.sql` ไฟล์เดียวจบ)
